The bet

Don't build another dashboard. Build the data layer.

Holdings spread across exchanges, on-chain wallets, and prediction markets means five tabs and ten logins every morning. The usual fix is yet another tracker UI. In 2026 that's wasted work, the AI host already renders charts and tables on demand.

The old way

Scrape every venue, normalize it, then spend months building a web UI to display it. Ship a dashboard that's outdated the day a new chain matters.

HeadlessTracker

Ship the clean, normalized, read-only data over MCP. Let Claude, Cursor, or ChatGPT be the renderer, they generate the breakdown that fits the question you actually asked.

What you can ask

Plain language in. Real positions out.

No SQL, no CLI to memorize. Each question maps to one or more MCP tools the host calls for you.

What do I own across everything?
get_holdings · unified across all accounts
How am I split between crypto and prediction markets?
get_allocations · by asset class, symbol, chain
How was this week, and what drove it?
get_pnl 7d + get_transactions
Risk check me, am I too concentrated?
risk-check prompt · scored PASS / WARN / ALERT
Walk me through my Polymarket bets by event.
get_polymarket_positions · FIFO realized P&L
Show my portfolio in Hungarian forint.
get_holdings currency=HUF · live FX
Interactive dashboard

Not just text, a live UI panel in the chat.

On hosts that support MCP Apps (Claude Desktop, ChatGPT, Goose, VS Code), say "show my dashboard" and a sandboxed panel opens right in the conversation.

  • →Three live tabs. Portfolio value + top positions, weekly 7-day delta, and a risk audit, each with its own charts.
  • →Currency switcher + refresh. Flip between USD / EUR / GBP / HUF; the panel makes its own follow-up tool calls.
  • →Live settings panel too. Add or remove accounts through a form, no terminal required, each validated before it's stored.
  • →Graceful fallback. No MCP Apps support? The same workflows run as preset prompts, same data, no live panel.

Connectors

Five data sources, one schema.

Every connector uses read-only credentials by design. Worst case if a key leaks is a portfolio read, never a fund movement.

Bybitread-only
V5 UNIFIED / SPOT / CONTRACT / FUND. Holdings + full transaction log.
Binanceread-only
Spot account plus optional Futures wallet and open positions with leverage and liquidation.
MetaMask / EVMread-only
6 chains on one Etherscan key. Multi-wallet, native + ERC-20 balances and transfers, custom token lists.
Solanapublic address
Native SOL + SPL tokens via public RPC and Jupiter price. Multi-wallet, no API key.
Polymarketpublic address
Positions + BUY/SELL trade history from the public data-api. Event-grouped, no API key.

Why local-first

Your keys never leave your machine.

This is a tool that reads money data. The security model is the whole point, so it's deliberately boring.

  • →Keys in your OS keychain. macOS Keychain, Linux Secret Service, Windows Credential Vault, via @napi-rs/keyring.
  • →Local SQLite cache. No server, no SaaS, no telemetry, no analytics pings. Nothing phones home.
  • →Read-only, no signing. There is no code path that can move your funds.
  • →Honest unknowns. Transferred-in tokens with no cost basis report null, never a fabricated $0.

Quick start

Running in about five minutes.

Requires Bun 1.3+. Credentials are validated and stored in your keychain at setup.

# 1. Install
npm i -g headless-tracker

# 2. Add an account (prompts for read-only credentials, then validates them)
headless-tracker setup bybit
headless-tracker list-accounts

# 3. Wire it into Claude Desktop — claude_desktop_config.json
{
  "mcpServers": {
    "headless-tracker": { "command": "headless-tracker" }
  }
}

# 4. Restart Claude Desktop and ask
#    "What's in my portfolio?"

Prefer to clone and run from source? The README has the git clone + bun install path and per-connector setup for all five integrations.
