## 2026-06-02 (kedd)

Shipped v1.0.1: made the published binary Node-runnable so `npx headless-tracker` works without Bun (bundled node-shebang ESM entry via `bun build --target=node`, plus a runtime SQLite adapter, bun:sqlite under Bun and node:sqlite under Node, zero native deps). Why it mattered: the published bin had a bun shebang and imported bun:sqlite, so npx detonated on the first line for anyone without Bun, which likely explains a lot of the 138-downloads / 0-issues silence. The non-obvious part: no single SQLite driver loads in both runtimes (bun:sqlite Bun-only, node:sqlite Node-only, Bun cannot load better-sqlite3's native addon, oven-sh/bun#4290), so the driver is picked at runtime, and node:sqlite being built in drops native deps to zero. Verified clean-room from the real npm registry under Node with Bun hidden from PATH (--help, list-accounts via node:sqlite, full MCP initialize + tools/list handshake). Pending: MCP registry listing (mcpName field added), README 7→15 tools drift.

Update (3rd session, ~19:00): dogfooded as user #1 and the connector setup flow failed on the first try, which is exactly why you dogfood. On a headless box (no OS Secret Service: Docker/WSL/server/CI), the keyring write fails and `setup` aborted right there, so the account never registered AND the vault's env-var fallback was unreachable (the data tools enumerate registered accounts, so an unregistered account is invisible regardless of env vars). Fix: extracted a shared `finalizeAccountSetup` (src/setup-finalize.ts) used by both the CLI and the setup_connector MCP tool, registers the account regardless of keychain result and on failure surfaces the exact HEADLESS_TRACKER_<CONNECTOR>_<ACCOUNT> env var to set; no secrets written to disk. Verified end-to-end on this keychain-less box via a Node harness against the real source: keyring fails, account registers, env-var creds resolve, live Solana fetch returns 13 holding rows. Added a hermetic unit test (320 tests now) + README "Headless / no OS keychain" section with per-connector JSON shapes. Shipped v1.0.4 (npm + registry live), confirmed published binary runs under Node. Flagged to Tomi: the env-var path puts creds in plaintext in the MCP config, so an opt-in encrypted file vault is a security-model decision worth weighing. Side note: `--version`/`-v` isn't a real flag yet (only `help` prints the version) — cheap follow-up. Tomorrow: decide on the file-vault question, wire `--version`, and keep watching installs vs registry discovery.
